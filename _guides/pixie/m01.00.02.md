## Declaring Variables

`VAR` declares a variable and requires an initial value to infer type:

```
VAR score = 0
VAR speed = 1.5
VAR title = "SNAKE"
```

Assignment afterward is just `=`:

```
score = score + 1
```

Pixie has two value types you'll use constantly - **numbers** (always floating point; `7 / 2` is `3.5`, not `3`) and **strings** (double quotes only, no escape sequences). Arrays and points get their own module ([M1.0.6](/guides/pixie/m01.00.06/)).

### One gotcha: don't re-declare

`VAR x = ...` compiles to JavaScript's `let x = ...`, so declaring the same name twice **in the same scope** is an error:

```
VAR x = 1
VAR x = 2      //prevents game from starting
```

The compile step will succeed but the game pane will stay blank, because the generated JavaScript fails as it loads. If you press **Run** and nothing appears, a duplicate `VAR` is the first thing to check. Declaring `VAR` inside a loop or `IF` body is fine as each block is its own scope.

## Arithmetic

The four operators you'd expect, plus parentheses and unary minus:

```
VAR cx = (left + right) / 2
VAR flipped = -dx
```

Precedence is standard: `*` and `/` bind tighter than `+` and `-`, comparisons bind loosest. There is **no modulo operator** - when you need "wrap around" behavior, use `FLOOR` division instead:

```
// wrapped = n mod 4
VAR wrapped = n - FLOOR(n / 4) * 4
```

## Comparisons

Six comparison operators, used mostly in `IF` conditions:

```
==   !=   <   >   <=   >=
```

```
IF score >= 10:
    tickRate = 5
END
```

There are **no `AND` / `OR` / `NOT` operators**. The idiomatic Pixie way to express "both must be true" is nesting:

```
// x == appleX AND y == appleY
IF x == appleX:
    IF y == appleY:
        score = score + 1
    END
END
```

... and "either may be true" is two separate `IF`s that do the same thing (or set a shared flag).

## Strings and `+`

`+` concatenates when a string is involved, which is how you build HUD text:

```
TEXT "SCORE: " + score AT 8 8
```

Numbers passed to `TEXT` are converted automatically, so `TEXT score AT 8 8` works too.

## Try It

A tiny program that exercises all of it - paste into the [editor](/apps/pixie):

```
GRID 160 160

VAR frames = 0

UPDATE:
    frames = frames + 1
    VAR seconds = FLOOR(frames / 60)

    CLEAR
    COLOR 40 40 60
    BOX 0 0 160 90

    COLOR 255 255 255
    FONT_SIZE 10
    TEXT "frames:  " + frames AT 10 20
    TEXT "seconds: " + seconds AT 10 40

    IF seconds >= 5:
        COLOR 255 200 80
        TEXT "five seconds!" AT 10 60
    END
END
```
