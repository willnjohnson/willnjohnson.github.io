Pixie is a tiny game-scripting language I built for making little pixel games in the browser. You write Pixie source, and a compiler turns it into JavaScript that draws on a canvas.

You don't need to install anything. Everything in this guide runs in the [Pixie editor](/apps/pixie). Paste code in, press **Run**, and the program plays right next to the source.

## The Shape of a Pixie Program

Pixie should be somewhat familiar, if you've ever programmed in a programming language such as Processing or Arduino.

Every Pixie program has two parts:

1. **Setup** - everything *outside* the `UPDATE:` block runs **once**, at startup. This is where you declare the grid and your variables.
2. **The `UPDATE:` block** - runs **every frame**, about 60 times per second, forever. This is your game loop: read input, move things, draw the scene.

```
GRID 128 128            // runs once: create a 128×128 pixel canvas

VAR ticks = 0           // runs once: declare state

UPDATE:                 // runs every frame
    ticks = ticks + 1
    CLEAR               // clear whatever was previous drawn
    COLOR 255 255 255   // set color to white
    TEXT ticks AT 10 26 // draws text at coordinates x: 10, y: 26
END
```

`GRID w h` must come before any drawing - it creates the canvas. The canvas is a *pixel grid*: whatever size you pick gets scaled up to fill the game pane with crisp, pixelated rendering, so a `GRID 64 64` game has big chunky pixels and a `GRID 320 320` game has fine ones.

## Hello World

The smallest Pixie program doesn't even need an `UPDATE:` block if an object is to be drawn once. Here's a simple "Hello World" program:

```
GRID 128 128

COLOR 255 255 255
FONT_SIZE 12
TEXT "Hello world!" AT 10 26
```

Paste that into the [editor](/apps/pixie), press **Run**, and you've written your first Pixie program.

A few things to notice:

- `COLOR r g b` sets the current draw color (0–255 per channel). Everything drawn afterward uses it, until the next `COLOR`.
- `TEXT something AT x y` draws text with its top-left corner at `(x, y)`. Coordinates start at the **top-left** of the grid; `y` grows *downward*.
- Commands take **space-separated** arguments - no commas, no parentheses.
- Comments start with `//` and run to the end of the line.

## Making The Text Move

Add an `UPDATE:` block and the same program becomes an animation:

```
GRID 128 128

VAR x = 0

UPDATE:
    x = x + 1
    IF x > 128:
        x = -80
    END

    CLEAR
    COLOR 255 255 255
    FONT_SIZE 12
    TEXT "Hello world!" AT x 26
END
```

The text scrolls across the screen and wraps around. That is the process: *update state, then redraw everything from scratch, every frame*. `CLEAR` wipes the canvas; without it each frame draws on top of the last (which is occasionally a fun effect, but usually a smear).
