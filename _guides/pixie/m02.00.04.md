2048 primarily focuses on the slide-and-merge rule: Each tile merges only once per move, and uses four directions which are just the same operation. We'll build it so that LEFT, RIGHT, UP, and DOWN share *one* implementation.

The complete, tested program is at the bottom. The sections walk through how each piece works; paste the full listing into the [editor](/apps/pixie) whenever you want to play along.

## State

```
GRID 308 368

VAR board = NEW_ARRAY(16)
VAR score = 0
VAR gameOver = 0
VAR needSpawn = 2
```

The board is the flat-array pattern: 4x4, index `r * 4 + c`. A cell holds `0` or a tile value (2, 4, 8, ...). The grid is 308x368: we use 308x308 for the board area with 60px for the header to display the score.

`needSpawn` does the following: Instead of duplicating "spawn a random tile" code for game-start and post-move, we keep a counter of *tiles owed*. It starts at 2 (a fresh game spawns two tiles), a successful move sets it to 1, and one block in `UPDATE:` pays the debt wherever it came from.

## Spawning

```
IF needSpawn > 0:
    FOR s FROM 1 TO needSpawn:
        VAR empties = ARRAY
        FOR i FROM 0 TO 15:
            IF board[i] == 0:
                ARR_PUSH empties i
            END
        END
        IF LEN(empties) > 0:
            VAR spawnIdx = empties[RANDOM(LEN(empties))]
            VAR spawnVal = 2
            IF RANDOM(10) == 0:
                spawnVal = 4
            END
            board[spawnIdx] = spawnVal
        END
    END
    needSpawn = 0
END
```

Collect the empty indices into an array, pick one with `RANDOM(LEN(empties))`, drop in a 2, or a 4, one time in ten. Collect-then-pick is a clean way to do "random empty cell".

## Directions Are Scan Patterns

Slide a row left:
- Read its tiles left-to-right
- Merge equal neighbors
- Write the result back left-to-right.

Slide right:
- Do the same thing, reading right-to-left. 

Sliding up and down:
- Ditto, reading along columns.

So a "direction" is nothing but *the order you visit cells*. Any of the four lines (rows or columns) can be described by a starting index and a step:

```
index of cell i in line l  =  l * lineStride + lineOffset + i * moveStep
```

| Key | `lineStride` | `lineOffset` | `moveStep` | Meaning |
|---|---|---|---|---|
| LEFT | 4 | 0 | 1 | rows, scanned →  |
| RIGHT | 4 | 3 | -1 | rows, scanned ← |
| UP | 1 | 0 | 4 | columns, scanned ↓ from top |
| DOWN | 1 | 12 | -4 | columns, scanned ↑ from bottom |
{: .pretty }

Input just records the pattern:

```
IF BTN LEFT:
    lineStride = 4
    lineOffset = 0
    moveStep = 1
    doMove = 1
END
```

... and one mover handles every direction. Four directions with zero duplicated logic.

## Slide and Merge

For each of the four lines, there are three passes:

**1. Collect** the non-zero tiles in scan order. Sliding is implicit, as the zeros simply don't come along:

```
VAR vals = ARRAY
FOR i FROM 0 TO 3:
    VAR cIdx = base + i * moveStep
    IF board[cIdx] > 0:
        ARR_PUSH vals board[cIdx]
    END
END
```

**2. Merge** adjacent equal pairs. For example, `[2, 2, 4]` must become `[4, 4]`, *not* `[8]`. A tile born from a merge can't merge again this move. We enforce it with a `skipNext` flag, so when a pair merges, we mark the next element as consumed:

```
VAR merged = ARRAY
VAR skipNext = 0
FOR i FROM 0 TO 3:
    IF i < LEN(vals):
        VAR doSkip = skipNext
        skipNext = 0
        IF doSkip == 0:
            VAR cur = vals[i]
            VAR nxt = 0
            IF i + 1 < LEN(vals):
                nxt = vals[i + 1]
            END
            IF cur == nxt:
                ARR_PUSH merged cur * 2
                score = score + cur * 2
                skipNext = 1
            END
            IF cur != nxt:
                ARR_PUSH merged cur
            END
        END
    END
END
```

Note the no-`ELSE` dance. We must copy the flag into `doSkip`, clear it, *then* test the copy. The element after a merge is skipped exactly once, and the flag is already clean for the one after that.

**3. Write back** in the same scan order, padding with zeros. Watch for change, because 2048 only spawns a new tile if the move actually moved something:

```
FOR i FROM 0 TO 3:
    VAR wIdx = base + i * moveStep
    VAR nv = 0
    IF i < LEN(merged):
        nv = merged[i]
    END
    IF board[wIdx] != nv:
        movedAny = 1
    END
    board[wIdx] = nv
END
```

If any line changed, `needSpawn = 1`, and the spawn block from earlier does the rest. A move that changes nothing (sliding left when everything's already packed left) spawns nothing, exactly like the real game.

## Game Over

You're stuck when there's no empty cell *and* no equal neighbors horizontally or vertically. Scan for any escape and set a flag:

```
VAR canMove = 0
FOR i FROM 0 TO 15:
    IF board[i] == 0:
        canMove = 1
    END
END
FOR r FROM 0 TO 3:
    FOR c FROM 0 TO 2:
        VAR hIdx = r * 4 + c
        IF board[hIdx] > 0:
            IF board[hIdx] == board[hIdx + 1]:
                canMove = 1
            END
        END
    END
END
// ... same scan down columns with board[vIdx + 4] ...
IF canMove == 0:
    gameOver = 1
END
```

The drawing section is long and unexciting, but we use the classic 2048 palette and a font size that shrinks as tile numbers grow digits.

## The Complete Game

```
//  2048  –  Pixie game
//  Board: 4x4 cells stored in a flat 16-slot array.
//  Controls: arrows / WASD to slide, any key to restart.

GRID 308 368

VAR board = NEW_ARRAY(16)
VAR score = 0
VAR gameOver = 0

// How many tiles to spawn this frame (2 at the start, 1 after a move)
VAR needSpawn = 2

// Direction of the pending move, encoded as a scan pattern:
// each of the 4 lines starts at  l*lineStride + lineOffset
// and steps by moveStep. Set by input, consumed by the mover.
VAR doMove = 0
VAR lineStride = 0
VAR lineOffset = 0
VAR moveStep = 0

UPDATE:

    // GAME OVER SCREEN
    IF gameOver == 1:
        CLEAR
        COLOR 250 248 239
        BOX 0 0 308 368

        COLOR 238 228 218 0.85
        BOX 0 60 308 308

        FONT_SIZE 26
        COLOR 119 110 101
        TEXT_CENTER "GAME OVER" AT 154 170

        FONT_SIZE 14
        TEXT_CENTER "SCORE  " + score AT 154 210

        FONT_SIZE 10
        COLOR 160 150 140
        TEXT_CENTER "PRESS ANY KEY" AT 154 250

        IF BTN ANY:
            board = NEW_ARRAY(16)
            score = 0
            needSpawn = 2
            gameOver = 0
        END
    END

    // ACTIVE GAMEPLAY
    IF gameOver == 0:

        // Input: pick a scan pattern for the slide
        IF BTN LEFT:
            lineStride = 4
            lineOffset = 0
            moveStep = 1
            doMove = 1
        END
        IF BTN RIGHT:
            lineStride = 4
            lineOffset = 3
            moveStep = -1
            doMove = 1
        END
        IF BTN UP:
            lineStride = 1
            lineOffset = 0
            moveStep = 4
            doMove = 1
        END
        IF BTN DOWN:
            lineStride = 1
            lineOffset = 12
            moveStep = -4
            doMove = 1
        END

        // Slide + merge all 4 lines toward the scan origin
        IF doMove == 1:
            doMove = 0
            VAR movedAny = 0

            FOR l FROM 0 TO 3:
                VAR base = l * lineStride + lineOffset

                // 1) Collect the line's non-zero tiles in scan order
                VAR vals = ARRAY
                FOR i FROM 0 TO 3:
                    VAR cIdx = base + i * moveStep
                    IF board[cIdx] > 0:
                        ARR_PUSH vals board[cIdx]
                    END
                END

                // 2) Merge adjacent equal pairs (each tile merges once)
                VAR merged = ARRAY
                VAR skipNext = 0
                FOR i FROM 0 TO 3:
                    IF i < LEN(vals):
                        VAR doSkip = skipNext
                        skipNext = 0
                        IF doSkip == 0:
                            VAR cur = vals[i]
                            VAR nxt = 0
                            IF i + 1 < LEN(vals):
                                nxt = vals[i + 1]
                            END
                            IF cur == nxt:
                                ARR_PUSH merged cur * 2
                                score = score + cur * 2
                                skipNext = 1
                            END
                            IF cur != nxt:
                                ARR_PUSH merged cur
                            END
                        END
                    END
                END

                // 3) Write the line back, padded with zeros
                FOR i FROM 0 TO 3:
                    VAR wIdx = base + i * moveStep
                    VAR nv = 0
                    IF i < LEN(merged):
                        nv = merged[i]
                    END
                    IF board[wIdx] != nv:
                        movedAny = 1
                    END
                    board[wIdx] = nv
                END
            END

            IF movedAny == 1:
                needSpawn = 1
            END
        END

        // Spawn tiles into random empty cells
        IF needSpawn > 0:
            FOR s FROM 1 TO needSpawn:
                VAR empties = ARRAY
                FOR i FROM 0 TO 15:
                    IF board[i] == 0:
                        ARR_PUSH empties i
                    END
                END
                IF LEN(empties) > 0:
                    VAR spawnIdx = empties[RANDOM(LEN(empties))]
                    VAR spawnVal = 2
                    IF RANDOM(10) == 0:
                        spawnVal = 4
                    END
                    board[spawnIdx] = spawnVal
                END
            END
            needSpawn = 0
        END

        // Draw
        CLEAR
        COLOR 250 248 239
        BOX 0 0 308 368

        FONT_SIZE 28
        COLOR 119 110 101
        TEXT "2048" AT 8 14

        FONT_SIZE 11
        COLOR 160 150 140
        TEXT "SCORE" AT 230 14
        FONT_SIZE 16
        COLOR 119 110 101
        TEXT score AT 230 30

        COLOR 187 173 160
        BOX 0 60 308 308

        FOR r FROM 0 TO 3:
            FOR c FROM 0 TO 3:
                VAR idx = r * 4 + c
                VAR v = board[idx]
                VAR px = 4 + c * 76
                VAR py = 64 + r * 76

                COLOR 205 193 180
                IF v == 2:
                    COLOR 238 228 218
                END
                IF v == 4:
                    COLOR 237 224 200
                END
                IF v == 8:
                    COLOR 242 177 121
                END
                IF v == 16:
                    COLOR 245 149 99
                END
                IF v == 32:
                    COLOR 246 124 95
                END
                IF v == 64:
                    COLOR 246 94 59
                END
                IF v == 128:
                    COLOR 237 207 114
                END
                IF v == 256:
                    COLOR 237 204 97
                END
                IF v == 512:
                    COLOR 237 200 80
                END
                IF v == 1024:
                    COLOR 237 197 63
                END
                IF v >= 2048:
                    COLOR 237 194 46
                END
                BOX px py 72 72

                IF v > 0:
                    VAR fs = 24
                    IF v >= 100:
                        fs = 20
                    END
                    IF v >= 1000:
                        fs = 16
                    END
                    FONT_SIZE fs

                    COLOR 119 110 101
                    IF v >= 8:
                        COLOR 249 246 242
                    END
                    TEXT_CENTER v AT px + 36 py + 36 - fs / 2
                END
            END
        END

        // Detect game over: no empty cell and no merge possible
        VAR canMove = 0
        FOR i FROM 0 TO 15:
            IF board[i] == 0:
                canMove = 1
            END
        END
        FOR r FROM 0 TO 3:
            FOR c FROM 0 TO 2:
                VAR hIdx = r * 4 + c
                IF board[hIdx] > 0:
                    IF board[hIdx] == board[hIdx + 1]:
                        canMove = 1
                    END
                END
            END
        END
        FOR c FROM 0 TO 3:
            FOR r FROM 0 TO 2:
                VAR vIdx = r * 4 + c
                IF board[vIdx] > 0:
                    IF board[vIdx] == board[vIdx + 4]:
                        canMove = 1
                    END
                END
            END
        END
        IF canMove == 0:
            gameOver = 1
        END
    END
END
```

## Now It's Your Turn
Create your own *pixie* project and see what you can build. This language is Turing complete, but it does have its own limitations (i.e. no way to define reusable procedures, no bitwise operators, no way to supply your own I/O, and performance is limited by the compiler-generated JS). *So, don't expect to build your own Gameboy emulator (or whatever ambitious project) with this language!*
