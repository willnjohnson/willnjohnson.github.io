Every block in Pixie - `UPDATE:`, `IF`, `FOR`, `FOR_EACH` - opens with a colon and closes with `END`. Indentation is readability; `END` is what the compiler reads.

## IF

```
IF score > highScore:
    highScore = score
END
```

There is **no `ELSE`**. Two patterns replace it. For small either/or logic, write both conditions:

```
IF caught == 1:
    score = score + 1
END
IF caught == 0:
    lives = lives - 1
END
```

For "compute a default, then override," assign first and let the `IF` overwrite:

```
VAR fill = 205        // default: empty-cell gray
IF v == 2:
    fill = 238        // override for a "2" tile
END
```

This second pattern shows up constantly in drawing code. Set the common case, override the special cases.

## FOR - counted loops

```
FOR i FROM 0 TO 9:
    BOX i * 16 0 15 15
END
```

Both endpoints are **inclusive** - `FROM 0 TO 9` runs ten times. To count down, use `DOWNTO`:

```
FOR row FROM 19 DOWNTO 1:
    // shift row-1 into row (how Tetris drops cleared lines)
END
```

Loops nest just fine, and nested loops over `x` and `y` are how you draw grids and boards:

```
FOR y FROM 0 TO 7:
    FOR x FROM 0 TO 7:
        VAR checker = x + y - FLOOR((x + y) / 2) * 2
        IF checker == 0:
            BOX x * 16 y * 16 16 16
        END
    END
END
```

(That `x + y - FLOOR(...) * 2` is the "mod 2" trick from [M1.0.2](/guides/pixie/m01.00.02/) as Pixie has no `%`.)

## FOR_EACH - array loops

```
FOR_EACH segment IN tail:
    BOX segment.x segment.y 16 16
END
```

Runs the body once per element, in order. This is the natural way to walk anything you built with arrays - snake tails, particle lists, tetromino blocks.

## No WHILE? No problem

Pixie has no `WHILE` loop, and for game code you almost never miss it: `UPDATE:` *is* your while loop, running every frame. When you genuinely need "repeat until a condition inside a single frame," run a `FOR` with enough iterations and guard the body with an `IF`. The Tetris hard drop does exactly this to slam a piece to the floor:

```
// descend one row per iteration until the next row collides;
// 20 iterations covers the full board height
FOR step FROM 1 TO 20:
    VAR collides = 0
    // ... set collides = 1 if the row below is blocked ...
    IF collides == 0:
        pieceY = pieceY + 1
    END
END
```

The loop always runs 20 times, but once the piece lands, the `IF` stops doing anything. Bounded loops like this are simple, safe (no infinite-loop risk), and plenty fast at Pixie's scale.

## State Machines with Flags

The most important control-flow *pattern* in Pixie isn't a keyword at all. Both built-in games structure their entire `UPDATE:` around a mode flag:

```
UPDATE:
    IF gameOver == 1:
        // draw the game-over screen; restart on keypress
    END

    IF gameOver == 0:
        // input, movement, collision, drawing
    END
END
```

One variable, two exclusive blocks: a two-state state machine. Add more flags (`paused`, `titleScreen`) and the same shape scales to any structure a small game needs. We'll lean on it in every Module 2 project.
