Module 1 covered the language itself. Now let's build things, starting with a ball that bounces off walls.

## Position + Velocity

Almost everything that moves in a game is two pairs of numbers: *where it is* (`x`, `y`) and *how it's moving* (`dx`, `dy`). Each frame you add the second pair to the first:

```
GRID 128 128

VAR x = 64
VAR y = 30
VAR dx = 1.5
VAR dy = 1

UPDATE:
    x = x + dx
    y = y + dy

    IF x < 5:
        dx = ABS(dx)
    END
    IF x > 123:
        dx = -ABS(dx)
    END
    IF y < 5:
        dy = ABS(dy)
    END
    IF y > 123:
        dy = -ABS(dy)
    END

    CLEAR
    COLOR 20 20 32
    BOX 0 0 128 128
    COLOR 255 200 80
    CIRCLE x y 5
END
```

Paste into the [editor](/apps/pixie) and run. The whole program is practically move, bounce, draw.

## Why `ABS` and Not Just Negate?

The tempting bounce is `dx = -dx`. It usually works, but if the ball ever ends up *past* the wall (say it moved 1.5px and overshot), plain negation can trap it. It flips direction, doesn't fully escape, flips again next frame, and buzzes in the wall forever.

Writing `dx = ABS(dx)` at the left wall says *"whatever you were doing, you are now moving right"*. This is the idempotent approach, so a ball two frames deep in the wall still comes out cleanly.

## Adding Gravity

Two more lines turn the bouncing ball into a ball that falls and loses energy:

```
GRID 128 128

VAR x = 30
VAR y = 20
VAR dx = 1.2
VAR dy = 0

UPDATE:
    dy = dy + 0.15             // gravity accelerates downward
    x = x + dx
    y = y + dy

    IF x < 5:
        dx = ABS(dx)
    END
    IF x > 123:
        dx = -ABS(dx)
    END
    IF y > 120:
        y = 120
        dy = -ABS(dy) * 0.85   // bounce, keeping 85% of the energy
    END

    IF BTN SPACE:
        dy = -4                // tap space to boot it upward
    END

    CLEAR
    COLOR 20 20 32
    BOX 0 0 128 128
    COLOR 90 90 110
    BOX 0 125 128 3
    COLOR 255 120 120
    CIRCLE x y 5
END
```

Velocity now *changes over time* (`dy = dy + 0.15`), which is all acceleration is. The `* 0.85` on the bounce bleeds off energy so the ball settles instead of bouncing forever. Finally, the `BTN SPACE` line sneaks in Module 1's input lesson: a one-shot tap that kicks the ball back up.

## Experiments to Try on Your Own

- Add friction (dx) when the ball touches the ground.
- Make the ball leave a trail: Delete `CLEAR` and draw the background with `COLOR 20 20 32 0.1` to create a translucent wipe that fades old frames instead of erasing them.
- Add a second ball with its own `x2/y2/dx2/dy2`.
