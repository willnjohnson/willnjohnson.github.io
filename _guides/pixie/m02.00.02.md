Time for a real game with a win/lose condition, score, and restart.

**Catch!**: Stars fall from the sky and you steer a tray to catch them. Miss three and it's over.

This project combines every pattern so far: the *steer-a-direction* input model ([M1.0.5](/guides/pixie/m01.00.05/)), nested-`IF` collision ([M1.0.2](/guides/pixie/m01.00.02/)), the game-over flag state machine ([M1.0.4](/guides/pixie/m01.00.04/)), and clamping with `MAX`/`MIN`.

## The Whole Game

```
GRID 160 160

VAR padX = 60
VAR padDx = 0
VAR starX = RANDOM(150)
VAR starY = 0
VAR speed = 1
VAR score = 0
VAR lives = 15
VAR gameOver = 0

UPDATE:
    IF gameOver == 1:
        CLEAR
        COLOR 10 10 22
        BOX 0 0 160 160

        FONT_SIZE 16
        COLOR 255 90 90
        TEXT_CENTER "GAME OVER" AT 80 55

        FONT_SIZE 10
        COLOR 230 230 230
        TEXT_CENTER "SCORE " + score AT 80 85
        COLOR 120 120 130
        TEXT_CENTER "PRESS ANY KEY" AT 80 110

        IF BTN ANY:
            score = 0
            lives = 15
            speed = 1
            starY = 0
            starX = RANDOM(150)
            padDx = 0
            gameOver = 0
        END
    END

    IF gameOver == 0:
        // steer the tray (taps set direction, motion persists)
        IF BTN LEFT:
            padDx = -2
        END
        IF BTN RIGHT:
            padDx = 2
        END
        padX = MAX(0, MIN(120, padX + padDx))

        // the star falls
        starY = starY + speed

        // reached the tray's row: catch or miss?
        IF starY > 144:
            VAR caught = 0
            IF starX + 8 > padX:
                IF starX < padX + 40:
                    caught = 1
                END
            END

            IF caught == 1:
                score = score + 1
                speed = speed + 0.02
            END
            IF caught == 0:
                lives = lives - 1
                IF lives == 0:
                    gameOver = 1
                END
            END

            starY = 0
            starX = RANDOM(150)
        END

        // draw
        CLEAR
        COLOR 10 10 22
        BOX 0 0 160 160

        COLOR 255 230 120
        CIRCLE starX + 4 starY 4

        COLOR 120 200 255
        BOX padX 148 40 6

        FONT_SIZE 9
        COLOR 255 255 255
        TEXT "SCORE " + score AT 4 4
        COLOR 255 120 120
        TEXT "LIVES " + lives AT 110 4
    END
END
```

Paste this code, run it, and tap left/right to steer.

## Notes

**The overlap test.** "Star overlaps tray" means *star's right edge is past the tray's left edge* AND *star's left edge is before the tray's right edge*. Because there is no `AND` in Pixie, it uses two nested `IF` statements setting a `caught` flag, then two flat `IF` statements acting on it. This is the standard shape for any either/or with real logic in both branches.

**Adding difficulty to the game:** The line `speed = speed + 0.2` runs on every catch, and the game gets a bit harder. This simple trick can create the sense of "difficulty" and it's what's used in Tetris.

**Restart resets *everything*:** Look at the `BTN ANY` block. It resets score, lives, speed, star, *and* `padDx`. Forgetting one (say, leaving `speed` at 5) is a classic restart bug, and can make the second run difficult to play. When you add states to a game, add its reset in the same breath.

## Experiment
