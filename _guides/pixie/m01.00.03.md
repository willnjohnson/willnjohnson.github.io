## The Canvas Model

Drawing in Pixie is *immediate*: each command paints pixels right away, in order, using the current color. There are no sprites or layers. You rebuild the whole picture every frame, back to front. Sky first, then the things in front of it, Bob Ross style.

```
GRID w h
```

Creates the pixel grid. Everything you draw is in grid coordinates: `(0, 0)` is the top-left corner, `x` grows right, `y` grows down.

## The Commands

```
CLEAR                      // wipe the canvas transparent
COLOR r g b                // set draw color, channels 0–255
COLOR r g b a              // ... with opacity 0–1
BOX x y w h                // filled rectangle, top-left at (x, y)
CIRCLE x y r               // filled circle centered at (x, y)
LINE x1 y1 x2 y2           // 1px line between two points
FONT_SIZE n                // text size in pixels (monospace)
TEXT value AT x y          // text, top-left anchored at (x, y)
TEXT_CENTER value AT x y   // text, horizontally centered on x
```

`COLOR` is *sticky* - it applies to every shape and text drawn after it, until changed. So group your drawing by color where you can:

```
COLOR 0 180 0
FOR_EACH segment IN tail:
    BOX segment.x segment.y 16 16
END
```

The fourth `COLOR` argument is alpha, from 0 (invisible) to 1 (opaque). It's how the built-in games dim the board behind a "GAME OVER" banner:

```
COLOR 0 0 0 0.65
BOX 28 85 200 100
```

`TEXT_CENTER` centers on the `x` you give it. Hand it the midpoint of your grid (`TEXT_CENTER "PAUSED" AT 64 60` on a 128-wide grid) and the text self-centers no matter how long it is. Both text commands accept numbers and string concatenations: `TEXT "HP " + hp AT 4 4`.

## Arguments Are Expressions

Any argument can be a full expression - this is what makes loops draw grids:

```
BOX BX + x * 16 BY + y * 16 15 15
```

**One parsing gotcha:** arguments are separated by spaces, and the parser reads each argument as greedily as it can. That's what lets `BX + x * 16` above count as *one* argument. But it means a negative literal after a variable gets absorbed into the previous argument:

```
BOX x -5 8 8       // parsed as BOX (x - 5) 8 8   i.e. 3 args, not 4!
BOX x (0 - 5) 8 8  // what you meant
```

When an argument needs to be negative, wrap it in parentheses.

## A Complete Scene

Here is a little nighttime scene using every command which you can paste into the [editor](/apps/pixie):

```
GRID 160 160

UPDATE:
    CLEAR

    // sky
    COLOR 15 15 40
    BOX 0 0 160 120

    // moon with a translucent halo
    COLOR 255 255 220 0.15
    CIRCLE 120 25 14
    COLOR 240 240 210
    CIRCLE 120 25 9

    // ground
    COLOR 20 60 30
    BOX 0 90 160 30

    // house
    COLOR 120 80 50
    BOX 30 60 40 30
    COLOR 255 220 120
    BOX 44 72 12 18

    // roof from two lines
    COLOR 90 60 40
    LINE 28 60 50 42
    LINE 50 42 72 60

    // caption
    COLOR 255 255 255
    FONT_SIZE 10
    TEXT_CENTER "good night" AT 80 105
END
```

Change the numbers, re-press **Run**, see what happens. That edit-run loop is the fastest way to get a feel for the coordinate system.
