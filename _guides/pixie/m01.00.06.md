## Arrays

Two ways to make one:

```
VAR tail = ARRAY            // empty, grows as you push
VAR board = NEW_ARRAY(200)  // 200 slots, all set to zero
```

`ARRAY` is for lists that grow and shrink (snake tails, particle effects). `NEW_ARRAY(n)` is for fixed-size storage you index into (game boards).

Reading, writing, and measuring:

```
VAR first = tail[0]
board[42] = 7
VAR count = LEN(tail)
```

And three commands that reshape a growable array:

```
ARR_PUSH tail value       // append to the end
ARR_UNSHIFT tail value    // insert at the front
ARR_POP tail              // remove from the end
```

`ARR_UNSHIFT` + `ARR_POP` together make a queue. Push the new head position onto the front, pop the oldest segment off the back, and you have a snake tail in two lines:

```
ARR_UNSHIFT tail POINT(x y)
IF LEN(tail) > tailLength:
    ARR_POP tail
END
```

## Points

`POINT(x y)` bundles two numbers into one value with `.x` and `.y` fields. Note the arguments are **space-separated**, not comma-separated:

```
VAR head = POINT(128 64)
TEXT head.x AT 4 4

FOR_EACH segment IN tail:
    BOX segment.x segment.y 16 16
END
```

Negative coordinates are fine. For instance, `POINT(-1 0)` is how Tetris describes tetromino block offsets relative to a piece's center.

## The Flat-Array Board

Pixie arrays are one-dimensional, but every grid game needs a 2D board. The universal trick is to store row-by-row in a flat array and convert coordinates with arithmetic.

```
// a 10-wide, 20-tall board in 200 slots
VAR board = NEW_ARRAY(200)

// (x, y)  ->  index
VAR idx = y * 10 + x

// index  ->  (x, y), using the no-modulo trick
VAR row = FLOOR(idx / 10)
VAR col = idx - row * 10
```

Both Tetris (10x20) and the 2048 we build in [M2.0.4](/guides/pixie/m02.00.04/) (4x4) live in flat arrays exactly like this. It's worth writing the little `y * width + x` expression until it's reflex, as it is *the* data structure of grid-based games.

## Math Built-ins

```
RANDOM(n)     // random integer from 0 to n-1
MAX(a, b)     // largest argument (two or more)
MIN(a, b)     // smallest argument
ABS(x)        // absolute value
FLOOR(x)      // round down
CEIL(x)       // round up
```

These built-ins use parentheses and **commas** like regular function calls. They are expressions, not commands. Three idioms cover most uses:

```
// snap a random position to a 16px grid
appleX = RANDOM(16) * 16

// clamp a paddle to the screen
padX = MAX(0, MIN(120, padX))

// speed up as score climbs, but never past a floor
tickRate = MAX(2, 10 - FLOOR(score / 3))
```

## Quick Reference

Everything in the language, one table:

| Category | Syntax |
|---|---|
| Setup | `GRID w h` |
| Variables | `VAR name = expr` · `name = expr` |
| Loop block | `UPDATE:` ... `END` |
| Conditionals | `IF cond:` ... `END` |
| Counted loop | `FOR i FROM a TO b:` / `FROM a DOWNTO b:` ... `END` |
| Array loop | `FOR_EACH item IN arr:` ... `END` |
| Drawing | `CLEAR` · `COLOR r g b [a]` · `BOX x y w h` · `CIRCLE x y r` · `LINE x1 y1 x2 y2` |
| Text | `FONT_SIZE n` · `TEXT v AT x y` · `TEXT_CENTER v AT x y` |
| Input | `BTN key` · `BTN_RELEASED key` |
| Arrays | `ARRAY` · `NEW_ARRAY(n)` · `arr[i]` · `LEN(a)` · `ARR_PUSH a v` · `ARR_UNSHIFT a v` · `ARR_POP a` |
| Points | `POINT(x y)` · `.x` · `.y` |
| Math | `RANDOM(n)` · `MAX` · `MIN` · `ABS` · `FLOOR` · `CEIL` |
| Operators | `+ - * /` · `== != < > <= >=` · `( )` · unary `-` |
{: .pretty }
