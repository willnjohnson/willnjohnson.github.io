## Two Input Expressions

Pixie reads the keyboard through two expressions, used anywhere a condition goes:

```
IF BTN LEFT:            // true on the frame the key is PRESSED
IF BTN_RELEASED LEFT:   // true on the frame the key is RELEASED
```

The key names:

| Name | Matches |
|---|---|
| `UP` `DOWN` `LEFT` `RIGHT` | Arrow keys **and** WASD |
| `SPACE` | Space bar |
| `ANY` | Any key at all |
| A letter, e.g. `BTN R` | That letter key |
{: .pretty }

For instance, you may use `ANY` for a game-over screen that reads "press any key to restart."

## One-Shot Semantics

Note: **`BTN` fires exactly once per press.** It's true on the single frame the key goes down, then false until the key is released and pressed again. Holding a key does *not* keep it true.

This enforces clean patterns:

**Pattern 1: taps do a thing.** For discrete actions (rotate, shoot, flap), `BTN` is exactly right as-is:

```
IF BTN SPACE:
    velocityY = -4
END
```

**Pattern 2: steer a persistent direction.** For continuous movement, don't move *while* the key is down. Use the tap to change a direction the game keeps applying. This is precisely how Snake works:

```
IF BTN LEFT:
    dx = -2
    dy = 0
END
IF BTN RIGHT:
    dx = 2
    dy = 0
END

x = x + dx      // motion continues every frame regardless
```

**Pattern 3: release-based taps.** `BTN_RELEASED` fires when a key comes *up*, which reads as a deliberate tap even if the player holds the key nervously. Tetris uses it for lateral moves and rotation so a held arrow can't auto-repeat a piece across the board.

## Timing Tricks: the Double-Tap

Because presses are one-shot events, you can build gesture-like input by recording *when* presses happen. Keep a frame counter, remember the last press, compare:

```
VAR frameClock = 0
VAR lastTap = -100

UPDATE:
    frameClock = frameClock + 1

    IF BTN DOWN:
        VAR gap = frameClock - lastTap
        IF gap < 36:
            // double tap! (two presses within ~0.6s)
        END
        IF gap >= 36:
            lastTap = frameClock
        END
    END
END
```

This is how the hard drop in Tetris is built. Two DOWN taps within 36 frames slams the piece to the floor. (`lastTap` starts at `-100` so the very first tap can never look like the second half of a double-tap.)

## Poll Every Frame

One subtlety worth internalizing early: **check `BTN` every frame, even if your game only "moves" every N frames.** Snake advances once per `tickRate` frames, but it polls input on *all* frames and buffers the requested direction:

```
IF BTN UP:
    IF dy == 0:
        nextDx = 0
        nextDy = -16
    END
END

// ...later, on tick frames only:
dx = nextDx
dy = nextDy
```

A press lands on one arbitrary frame; if you only looked at `BTN` on tick frames, most presses would fall in the gaps and vanish. Buffer on any frame and apply on the tick. No press is ever lost.

## Try It

```
GRID 128 128

VAR x = 60
VAR y = 60
VAR dx = 0
VAR dy = 0

UPDATE:
    IF BTN UP:
        dx = 0
        dy = -1
    END
    IF BTN DOWN:
        dx = 0
        dy = 1
    END
    IF BTN LEFT:
        dx = -1
        dy = 0
    END
    IF BTN RIGHT:
        dx = 1
        dy = 0
    END
    IF BTN SPACE:
        dx = 0
        dy = 0
    END

    x = MAX(0, MIN(120, x + dx))
    y = MAX(0, MIN(120, y + dy))

    CLEAR
    COLOR 25 25 35
    BOX 0 0 128 128
    COLOR 120 220 255
    BOX x y 8 8
END
```

Tap an arrow to glide and space to stop. Remember to click the game pane first so it receives your keys.
