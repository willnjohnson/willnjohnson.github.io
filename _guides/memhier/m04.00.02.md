This final module is about making your simulator trustworthy.

## Common Implementation Mistakes

### Wrong Address Breakdown

- Using the raw trace address for cache tag when virtual translation is enabled
- Using the wrong number of offset or index bits
- Not preserving the page offset during translation

### Cache Fill Mistakes

- Filling L1 on a write miss with no-write-allocate
- Forgetting to write back dirty lines in write-back mode
- Not invalidating L1 lines when an inclusive L2 line is evicted

### Page Replacement Mistakes

- Evicting the wrong physical page
- Failing to invalidate TLB entries for the evicted page
- Counting an extra disk reference or missing one

### Statistics Mistakes

- Incrementing `mem_refs` for every cache miss
- Counting `l2_hits` even when the L1 path took the result
- Failing to increment `pt_refs` when the page table is accessed

## Verifying Correctness

### Step-by-step Trace Printout

Print a line for each reference with:

- Input address
- VPN and offset
- TLB result
- PT result
- Physical page
- Cache tags, indexes, and hit/miss

Module 1.2 has the exact field order and printf layout the assignment expects. If your output columns are shifted or blank in the wrong spot, that's usually the fastest way to spot where the logic diverges before you even look at hit/miss correctness.

### Regression Test Cases

Create small traces that isolate behavior:

- Repeated access to the same line
- Sequential accesses across page boundaries
- Write-allocate vs no-write-allocate
- L2 hit after L1 miss
- Page fault after physical memory is full

### Confirm Layer Order

Ask yourself for each access:

- Did translation happen before cache lookup?
- Did the simulator consult L2 only after L1 misses?
- Did dirty write-back happen in the correct direction?

## Programming Tips

- Use `printf` in the early stages to verify field values
- Temporarily make the simulator verbose while debugging
- Limit the trace to 10-20 lines when validating
- Write a helper to dump one set of cache tags and valid bits

## Final Correctness Checklist

- Raw trace is parsed correctly
- Address decomposition math is right
- TLB and page table behavior are distinct
- L1 and L2 lookups are separate
- Write policies are implemented correctly
- Evictions update lower levels and statistics

## What This Means

A correct simulator is not just a working program.

It is a program where each stage has a clear contract, each counter has a clear trigger, and the path from trace input to final statistics is easy to follow.
