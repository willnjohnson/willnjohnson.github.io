Replacement is the part of the simulator that decides what gets thrown away.

## Why LRU is a Good First Choice

Though there are alternatives such as FIFO, Random, Second Chance, and LFU, the assignment expects true LRU.

- Simplest to understand
- Matches the replacement behavior described by the assignment description
- Easy to implement with age counters

## Replacement in Caches

Each set holds N ways.

When a miss occurs:

- If there is an invalid line, use it
- Otherwise, evict the line with the **highest age**

### LRU Update Rule

On every access:

- Increment age for every valid line in the set
- Reset the accessed line's age to zero

This is simpler than maintaining a linked list.

## Page Frame Replacement

The page table uses the same idea.

When physical memory is full:

- Choose the physical page with the highest age
- Invalidate any cache or TLB entries using that page
- If the evicted page was dirty, count a disk write

## Implementation Diagram

```
for each line in set:
  if valid: line.age++
  if line.tag == requested_tag: hit

if hit:
  line.age = 0
else:
  victim = find_max_age_or_invalid()
  replace victim
```

## Why Eviction Order Matters

If you evict the wrong line:

- You may undercount cache hits
- Dirty lines may not be written back correctly
- Inclusivity between L1 and L2 may break

## Programming Tips

- Keep the LRU logic inside the cache lookup helper
- Return the victim line along with hit/miss
- If you use a fixed-size array, you can compute the victim index directly
- Initialize ages to zero and valid bits to false

## Common Mistakes

- Resetting only the victim's age on miss but not the hit age on hit
- Using LRU ages as timestamps without clearing them
- Evicting the wrong line when all lines are valid
- Not updating the age for all lines in the set

## What to Practice

- Implement `find_victim_line()` separately from `cache_lookup()`
- Test with direct-mapped cache first (`assoc = 1`)
- Test with 2-way and 4-way associativity
- Check that a repeated reference to the same line returns hit after the first access

This module should make replacement logic feel like a clean, reusable rule.
