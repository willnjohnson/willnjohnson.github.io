## Introduction

You're building a book inventory manager: a program that tracks title, author, ISBN, and a cover image URL for a collection of books, and lets you create, read, update, and delete entries. This module sets up the project skeleton.

## Project Structure

```tree
book-inventory/
 data/
  books.json
 src/
  inventory.js
  storage.js
  cli.js
 index.js
 package.json
```

- `storage.js`: the only file that touches the filesystem. Reads and writes `data/books.json`.
- `inventory.js`: pure CRUD logic that operates on data already in memory.
- `cli.js` / `index.js`: wires the two together and gives you something to run.

## Setting Up the Project

```
mkdir book-inventory && cd book-inventory
npm init -y
mkdir data src
echo [] > data/books.json
```

No external dependencies are needed yet, and Node's built-in `fs` module is enough to read and write JSON.

## Where to Start Coding

Before writing any CRUD logic, get `storage.js` loading and saving the JSON file:

```js
// src/storage.js
const fs = require("fs");
const path = require("path");

const DATA_FILE = path.join(__dirname, "..", "data", "books.json");

function loadRaw() {
  const text = fs.readFileSync(DATA_FILE, "utf8");
  return JSON.parse(text);
}

function saveRaw(rawArray) {
  fs.writeFileSync(DATA_FILE, JSON.stringify(rawArray, null, 2));
}

module.exports = { loadRaw, saveRaw };
```

Wire this into `index.js` and print the loaded array immediately:

```js
// index.js
const { loadRaw } = require("./src/storage");
console.log(loadRaw());
```

Run it. If you see `[]` printed back, then persistence works.

## Programming Tips

- Keep `storage.js` limited to reading and writing the whole file. Don't let it grow "find one book" or "update one field" logic, as that belongs in `inventory.js`.
- Use `JSON.stringify(data, null, 2)` so `books.json` stays readable in a diff, not a single unreadable line.
