## Introduction

This guide walks through building three small, real applications from scratch. Each one introduces a domain you'll run into constantly as a developer, and each one is scoped to be completed.

## What You'll Build

- **Module 1: Book Inventory (JavaScript):** A CRUD app for managing a book collection: define the data, implement create/read/update/delete, then wire it to a browser front-end.
- **Module 2: Authentication Dashboard:** Password hashing, session management, role-based access, and TLS client certificates for privileged routes.
- **Module 3: Realtime Chat Server (Rust):** A TCP/WebSocket server that handles concurrent client connections, exposes an HTTP API, and keeps chat history.

## Why These Three Projects

Module 1 teaches you to model data and mutate it safely. Module 2 layers identity on top of that: now that you can store data, who's allowed to read or change it? Module 3 explores allowing multiple clients talk to a program at the same time, and you have to get concurrency right instead of debugging it after the fact.

- **Module 1:** Node.js installed, and knowledge of basic JS (variables, functions, arrays and objects).
