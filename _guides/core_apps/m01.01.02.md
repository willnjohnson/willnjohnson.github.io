## Introduction

Before writing a single CRUD function, decide two things: what a "book" looks like in code, and how the collection of books is stored and looked up.

## The Book Object

```js
const book = {
  isbn: "978-0134685991",
  title: "Effective Java",
  author: "Joshua Bloch",
  image: "https://covers.openlibrary.org/b/isbn/9780134685991-L.jpg",
  addedAt: "2026-07-19T00:00:00.000Z"
};
```

Use the ISBN as the unique identifier. It's already a real-world identifier assigned to the book, it's stable, and helps with looking up an exact title since multiple books can share the same title, but not the same ISBN.

## Array vs. Map: Choosing How to Store the Collection

You have two options for the in-memory collection:

- **Array of book objects**: simple to serialize straight to JSON, easy to iterate for Read/List. But Create, Update, and Delete by ISBN each need an `O(n)` scan to find the right entry.
- **`Map` keyed by ISBN**: Create, Update, and Delete become `O(1)` lookups. List just means iterating the values.

Use a `Map` in memory for CRUD, and convert to a plain array only when you serialize to `books.json`:

```js
function loadInventory(rawArray) {
  const inventory = new Map();
  for (const book of rawArray) inventory.set(book.isbn, book);
  return inventory;
}

function serializeInventory(inventory) {
  return Array.from(inventory.values());
}
```

This keeps the on-disk shape (a plain array, easy for any tool to read) separate from the in-memory shape (a `Map`, fast to mutate). `inventory.

## Validating a Book Shape

A minimal check before anything gets inserted:

```js
function isValidBook(book) {
  return (
    typeof book?.isbn === "string" && book.isbn.length > 0 &&
    typeof book?.title === "string" && book.title.length > 0 &&
    typeof book?.author === "string" && book.author.length > 0
  );
}
```

`image` is left optional here; a placeholder image cover can be defaulted.

## Programming Tips

- Treat ISBN as an opaque string, never a number. Leading zeros and hyphens are meaningful, and `parseInt` will silently corrupt them.
- Fix the on-disk shape once and don't let the in-memory `Map` shape leak into `storage.js`.
